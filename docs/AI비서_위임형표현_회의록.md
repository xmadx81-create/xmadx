## 1. 개회 및 문제 현황

**전현무:** 여러분, 앞선 버그 분석 회의에 이어 바로 이어갑니다. 이번에는 사용자가 AI 비서에게 "알아서해줘", "일해줘" 같은 위임형 표현을 사용했을 때 비서가 전혀 알아듣지 못하는 문제입니다. 스크린샷 4장이 제보됐는데, 하나하나가 다 심각해요.

**전현무:** 현재 파악된 실패 사례입니다.

> **사례 1**: 사용자 "알아서해줘" → AI "무엇을 도와드릴까요? 구체적으로 말씀해주세요!" (기대: 현재 할일·일정·출결 종합 브리핑)  
> **사례 2**: 사용자 "일해줘" → AI 동일한 제네릭 응답 (기대: 미처리 업무 자동 분석)  
> **사례 3**: 사용자 "내가 뭘해야하는지 니가 알아야지" → AI "도움말이라고 하시면..." (기대: AI가 사용자 상황 파악 후 우선순위 안내)  
> **사례 4**: 사용자 "아니 니가 알아야한다고" → AI "이해하기 어려웠어요" (완전 폴백 — 기대: 재시도 자동 브리핑)

**전현무:** 사용자가 네 번이나 같은 의도로 말했는데 AI가 한 번도 알아듣지 못했습니다. 이건 단순 버그가 아니라 설계 공백이에요. 자연어처리 파트, 원인 분석 발표해주세요.

## 2. 원인 분석 — 자연어처리 파트 발표

**김지은 선임:** 자연어처리 파트입니다. 원인은 명확합니다. app.js 9708번 라인의 스마트 추천 트리거 정규식이 문제입니다. 현재 패턴은 이렇게 되어 있어요.

```
/추천해줘|뭐\s*하면\s*좋|지금\s*뭐\s*할|할\s*일.*추천|뭐\s*할까/
```

**김지은 선임:** 보시다시피, "추천해줘", "뭐 하면 좋을까", "지금 뭐 할까" 같은 **명시적 요청**만 매칭합니다. 한국어에서 가장 자연스럽게 쓰이는 위임형 표현 — "알아서해줘", "일해줘", "니가 판단해" — 이런 건 패턴에 아예 존재하지 않습니다.

**이준호 주임:** 제가 정규식을 전수 검사했는데, "알아서"라는 단어가 app.js 전체에서 단 한 번도 등장하지 않습니다. "일해줘"도 마찬가지고요. 이건 처음부터 고려되지 않은 겁니다.

**배성재:** 잠깐, 제가 하나 짚고 넘어가도 될까요? 이건 언어학에서 말하는 **간접 화행(indirect speech act)** 문제입니다. "알아서해줘"라는 문장의 표면적 의미는 "네가 알아서 하라"는 명령이지만, 실제 화행은 "내 상황을 파악해서 적절한 정보를 줘"라는 요청이에요. 한국어는 이런 간접 화행이 특히 많은 언어입니다.

**김지은 선임:** 정확합니다. 그리고 현재 정규식은 **직접 화행**만 처리합니다. "추천해줘"는 직접 화행이니까 잡히고, "알아서해줘"는 간접 화행이라 빠지는 거죠.

**배성재:** "일해줘"도 마찬가지예요. 표면적으로는 "일하라"는 명령이지만, 실제 의도는 "네가 할 수 있는 일을 찾아서 브리핑하라"는 뜻이죠. 한국 직장 문화에서 상사가 비서에게 "일 좀 해"라고 하면, 비서가 "무슨 일을 할까요?"라고 되묻는 게 아니라 알아서 서류 정리하고 일정 챙기고 하잖아요.

**이경규:** *(강하게)* 그거야! 비서가 "뭘 해드릴까요?"라고 물어보면 그건 비서가 아니야! 내가 30년 방송하면서 매니저한테 "알아서 해"라고 하면, 그 매니저가 "뭘 알아서 할까요?"라고 되물었으면 그날 잘렸어. 진짜 비서는 알아서 하는 거야!

**강호동:** 맞아, 맞아. 나도 촬영장에서 스텝한테 "알아서 해!"라고 하면 다들 알아서 하거든. 카메라 세팅하고, 마이크 체크하고, 대본 준비하고. 누가 "뭘 알아서 할까요?"라고 묻냐고!

**전현무:** 좋습니다. 원인이 명확하네요. 정규식에 위임형 표현 패턴이 아예 없다. 그리고 이건 단순히 패턴 몇 개 추가하는 문제가 아니라, 한국어 간접 화행을 이해하는 구조적 개선이 필요하다는 거죠.

## 4. 초대 손님 의견 종합

**이경규:** 그리고 한 가지 더. 같은 말을 두 번 했는데 AI가 못 알아들으면, 세 번째에는 무조건 브리핑을 때려야 해. 사용자가 반복한다는 건 "야 제발 좀 알아들어라"라는 뜻이거든. 세 번째에도 "구체적으로 말씀해주세요"라고 하면 그건 벽한테 말하는 거야.

**강호동:** 나는 이런 것도 있었으면 좋겠어. 사용자가 아무것도 안 물어봐도, AI가 먼저 "오늘 이런 거 있으시네요" 하고 알려주는 거. 아침에 앱 켜면 바로 "오늘 회의 2건, 미처리 업무 3건 있습니다" 이렇게.

**기안84:** 근데 그러면 AI가 너무 나대는 거 아니에요? 안 물어봤는데 자기가 먼저 떠드는 건... *(고민하며)* 근데 비서니까 그래도 되나?

## 5. 개선안 논의 — 구체적 해결 방안

### A. 위임형 표현 패턴 추가 (30종+)

**김지은 선임:** 배성재씨가 정리해주신 5가지 범주를 기반으로, 정규식에 추가할 패턴 목록을 작성했습니다.

```
// 위임형 표현 감지 정규식 (30종+)
const delegationPattern = new RegExp([
  // 1. 포괄적 위임
  '알아서\\s*해', '알아서\\s*해줘', '니가\\s*알아서', '너가\\s*알아서',
  '알아서\\s*척척', '알아서\\s*잘', '알잘딱', '맡길게',
  // 2. 업무 투척
  '일해줘', '일\\s*좀\\s*해', '일하라고', '일\\s*시작',
  '일거리', '할\\s*거\\s*줘',
  // 3. 판단 위임  
  '니가\\s*알아야지', '니가\\s*판단', '너가\\s*정해',
  '니가\\s*결정', '너한테\\s*맡', '네가\\s*알아',
  // 4. 상황 질의
  '뭘\\s*해야\\s*돼', '뭐\\s*해야\\s*해', '뭘\\s*해야\\s*하지',
  '뭐\\s*하지', '해야\\s*할\\s*게\\s*뭐', '할\\s*게\\s*뭐',
  // 5. 자동화 요청
  '시켜줘', '자동으로\\s*해', '자동\\s*브리핑',
  '알아서\\s*정리', '정리해줘', '쭉\\s*정리'
].join('|'), 'i');
```

**이준호 주임:** 이 패턴에 매칭되면 기존 `_aiSmartRecommend()` 함수를 호출하거나, 더 포괄적인 `_aiNextAction()` 함수를 실행해서 사용자 데이터 기반 브리핑을 제공합니다.

**전현무:** 30종이면 충분합니까?

**김지은 선임:** 초기 커버리지로는 충분합니다. 다만 한국어 특성상 변형이 무한하기 때문에, 뒤에서 논의할 반복 질문 감지와 맥락 추론을 함께 적용해야 합니다.

### B. 맥락 기반 자동 브리핑 (시간대+할일+일정+출결 종합)

**최민수 선임:** 예측/분석 파트입니다. 위임형 표현이 감지되면, 단순히 "추천" 하나만 주는 게 아니라 **종합 브리핑**을 제공해야 합니다. 시간대별로 내용이 달라져야 하고요.

**한지원 주임:** 구체적으로 이런 구조입니다.

| 시간대 | 브리핑 내용 |
|--------|-------------|
| 출근 시간 (07~09시) | 오늘 일정 + 미처리 할일 + 출결 현황 + 어제 미완료 업무 |
| 오전 업무 (09~12시) | 오전 중 마감 업무 + 다가오는 일정 알림 |
| 점심 (12~13시) | 오후 일정 미리보기 + 오전 처리 현황 |
| 오후 업무 (13~18시) | 오늘 마감 업무 + 남은 할일 + 퇴근 전 체크리스트 |
| 퇴근 후 (18시~) | 오늘 업무 요약 + 내일 예정 미리보기 |

**최민수 선임:** 사용자가 "알아서해줘"라고 하면, 현재 시간대에 맞는 브리핑을 자동으로 구성해서 보여주는 겁니다. 이미 `_aiSmartRecommend()` 함수에 할일, 일정, 출결 데이터를 조회하는 로직이 있으니까, 시간대 분기만 추가하면 됩니다.

**최민수 선임:** 네, 정확히 그렇습니다.

### C. "해줘" 단독 입력 시 맥락 추론 (이전 대화 기반)

**강현우 선임:** 기억/학습 파트입니다. "해줘"만 단독으로 입력되는 경우가 있습니다. 이건 너무 범용적이라 단독으로는 위임형으로 판단하기 어렵습니다.

**송미래 주임:** 그래서 이전 대화 맥락을 참조해야 합니다. 예를 들어, 직전에 "보고서 써야 하는데..."라고 했으면, 바로 다음에 "해줘"라고 하면 보고서 작성을 의미하는 거죠. 직전 대화가 없거나 맥락이 불분명할 때만 종합 브리핑으로 폴백합니다.

**강현우 선임:** 구현 방식은 이렇습니다. 대화 히스토리에서 최근 3개 메시지를 확인하고, 거기서 업무 키워드(보고서, 일정, 할일, 출결 등)가 있으면 해당 기능으로 연결합니다. 없으면 `_aiSmartRecommend()`로 종합 브리핑을 제공합니다.

**기안84:** 그러니까 "해줘"만 치면 AI가 "아까 보고서 얘기했으니까 보고서 쓸까?" 이런 거예요? 그건 좀 똑똑하긴 한데... 만약에 제가 아까 라면 얘기를 했으면 "라면 끓여드릴까요?" 이러는 건 아니죠?

*(전원 웃음)*

**송미래 주임:** 업무 관련 키워드만 인식합니다. "라면"은 업무 키워드가 아니므로 종합 브리핑으로 넘어갑니다.

### D. 반복 질문 감지 (같은 의미 2번 이상 → 자동 브리핑)

**박소영 선임:** 감정/인격 파트입니다. 스크린샷 사례 3, 4번을 보면, 사용자가 "내가 뭘해야하는지 니가 알아야지" → "아니 니가 알아야한다고"로 같은 의도를 반복합니다. 이때 AI가 두 번째에도 동일한 폴백을 보여주면 사용자 감정이 급격히 나빠집니다.

**정다은 주임:** 감정 점수 추적 데이터를 보면, 같은 의미를 반복했을 때 두 번째 폴백에서 사용자 좌절감이 270% 증가합니다. 세 번째가 되면 앱 이탈률이 85%까지 올라가고요.

**박소영 선임:** 개선안은 이렇습니다. 미인식 입력이 연속 2회 이상 들어오면, 그리고 그 입력들이 의미적으로 유사하면(같은 의도의 반복), 세 번째 시도를 기다리지 않고 **두 번째에서 즉시 종합 브리핑을 트리거**합니다. "정확히 이해하지 못했지만, 현재 상황을 정리해드릴게요"라고 하면서 자동으로 브리핑을 시작하는 거죠.

**이경규:** 그게 맞아! 두 번까지는 봐준다. 세 번째에 또 못 알아들으면 그건 진짜 벽이야. 근데 한 번 더 나아가서, 사용자가 짜증 내는 게 감지되면 — "아니", "아니라고", "니가"처럼 강조 표현이 들어오면 — 바로 브리핑 모드로 전환해야 해.

**정다은 주임:** 네, 감정 엔진에서 **좌절 감지** 패턴도 함께 추가하겠습니다. "아니", "아니라고", "그게 아니라", "말을 안 듣네" 같은 표현이 오면 좌절 플래그를 올리고, 즉시 맥락 전환을 시도합니다.

### E. AI 자발적 제안 강화 (사용자가 막연해 할 때 먼저 안내)

**오세준 선임:** UX/UI 파트입니다. 강호동씨가 제안하신 내용과 관련됩니다. 사용자가 앱을 켜고 아무 입력 없이 10초 이상 머무르거나, "음...", "뭐지", "어..." 같은 막연한 입력을 하면, AI가 먼저 간단한 현황 요약을 제시하는 겁니다.

**류하나 주임:** 다만 기안84씨가 우려하신 것처럼, AI가 너무 적극적이면 사용자가 부담을 느낄 수 있습니다. 그래서 **레벨제**를 제안합니다.

| 레벨 | 조건 | AI 행동 |
|------|------|---------|
| Lv.1 소극적 | 앱 진입 후 10초 무입력 | 하단에 "오늘 일정 3건이 있어요" 작은 배너 표시 |
| Lv.2 보통 | 막연한 입력 ("음...", "뭐지") | "현재 상황을 정리해드릴까요?" 제안 버튼 표시 |
| Lv.3 적극적 | 위임형 표현 감지 | 즉시 종합 브리핑 실행 |
| Lv.4 긴급 | 반복 미인식 + 좌절 감지 | "죄송합니다. 바로 현황을 정리해드릴게요" + 자동 브리핑 |

**오세준 선임:** 이렇게 하면 기안84씨가 말씀하신 "AI가 너무 나대는" 문제도 피하면서, 강호동씨가 원하시는 "알아서 하는 비서" 경험도 제공할 수 있습니다.

**기안84:** 오... Lv.1은 조용하게 알려주는 거니까 괜찮겠다. 레벨이 올라갈수록 적극적인 거고. 이건 좋네요.

**박지훈:** 저는 기본 세팅이 Lv.3이었으면 좋겠는데요. MZ세대는 AI한테 "알잘딱깔센"을 기대하는 거잖아요. 처음부터 적극적으로 해줘야 "오 이거 쓸만하다" 하고 계속 쓰지, Lv.1부터 시작하면 "이거 뭐야 조용하기만 하네" 하고 안 쓸 것 같은데...

**류하나 주임:** 사용자 설정으로 기본 레벨을 선택할 수 있게 하면 양쪽 다 만족시킬 수 있습니다.

### F. 추가 제안

**배성재:** 하나 더 제안합니다. "브리핑"이라는 정확한 키워드는 이미 작동하잖아요. 그런데 그 캐주얼한 동의어들은 안 됩니다. "요약해줘", "정리해줘", "상황 알려줘", "뭐 있어?", "오늘 뭐 있냐" 같은 것도 전부 브리핑 트리거에 포함시켜야 합니다.

**이준호 주임:** 동의합니다. "브리핑" 키워드의 동의어 확장도 함께 진행하겠습니다.

**백무결 대리:** 한 가지 더요. 위임형 표현으로 브리핑이 나올 때, 마지막에 "이 중에서 바로 처리할 항목이 있으면 번호를 눌러주세요" 같은 액션 버튼이 있으면 좋겠어요. 브리핑만 보여주고 끝나면 또 뭘 해야 할지 고민해야 하니까.

**오세준 선임:** 좋은 의견입니다. 브리핑 결과에 **원클릭 실행 버튼**을 추가하는 건 UX적으로도 자연스럽습니다.

**강호동:** 나 하나만 더! 사용자가 "알아서해줘"라고 했는데 할 일이 아무것도 없으면 어떻게 해? "알아서 쉬세요"라고 해? *(웃음)*

**최민수 선임:** 데이터가 전부 비어있으면 "현재 대기 중인 업무가 없습니다. 새로운 할일을 등록하시겠어요?"로 안내합니다. "알아서 쉬세요"는... 고려해보겠습니다.
