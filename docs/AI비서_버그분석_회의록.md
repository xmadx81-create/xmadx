### 버그 3 분석: "이게 일지라고?" 불만 → 보고서 작성 트리거

**김지은 선임:** 자연어처리 파트입니다. 이건 app.js 9045번 라인의 정규식 문제입니다. 현재 패턴이 `/보고서|일지|업무\s*일지|쓸래/`로 되어 있는데, "이게 일지라고?"라는 문장에서 "일지"가 매칭되면서 보고서 작성 프롬프트가 뜬 겁니다.

**이준호 주임:** 핵심은 **맥락 무시**입니다. "일지 써줘"와 "이게 일지라고?"는 의미가 완전히 다른데, 현재 정규식은 키워드만 잡기 때문에 구분을 못 합니다. "~라고?", "이게 ~", "뭐야 이게" 같은 부정적 맥락 패턴을 제외 조건으로 넣어야 합니다.

**김지은 선임:** 구체적으로, 현재 정규식 앞에 부정 맥락 필터를 추가해야 합니다. 예를 들어:

```javascript
// 부정 맥락 패턴
const negativeContext = /이게\s*.+라고|.+라고\?|뭐야\s*이게|아닌데|이런\s*게/;
// 기존 패턴 매칭 전에 부정 맥락 체크
if (negativeContext.test(userInput)) {
    // 보고서 트리거 스킵 → 불만 인식 로직으로 분기
}
```

**배성재:** 언어학적으로 보면 이건 전형적인 **화행(speech act) 오분류** 문제예요. "일지 써줘"는 지시(directive) 화행이고, "이게 일지라고?"는 표현(expressive) 화행, 구체적으로는 불만/항의입니다. 키워드 매칭만으로는 화행 구분이 불가능합니다. 최소한 문장 끝 종결어미와 부정 표현 조합을 봐야 합니다.

**전현무:** 배성재 씨답게 정확한 분석이네요. 그러니까 단어만 보지 말고 문장 전체 맥락, 특히 종결 표현을 봐야 한다는 거죠.

### A. 위자드 제출 전 확인 단계 추가

**강현우 선임:** app.js 10700번 라인의 `submitReport()` 호출 전에 미리보기 화면을 삽입합니다. 구체적으로:

1. 위자드 마지막 단계 완료 시 바로 제출하지 않고, 완성된 보고서 전문을 채팅 창에 카드 형태로 표시
2. 카드 하단에 **"이대로 제출할까요?"** / **"수정하기"** / **"취소"** 버튼 3개 배치
3. "수정하기" 선택 시 마지막 단계로 돌아감
4. "취소" 선택 시 위자드 전체 초기화 + "보고서 작성을 취소했어요" 안내

### B. 보고서/일지 패턴 매칭 개선

**김지은 선임:** 9045번 라인의 정규식을 개선합니다:

1. **부정적 맥락 필터 추가**: "이게 ~라고?", "~라고?", "뭐야 이게", "아닌데", "이런 게" 등의 패턴이 감지되면 보고서 트리거를 스킵
2. **요청형 종결어미 필수 매칭**: "~써줘", "~쓸래", "~쓰자", "~작성해줘" 등이 함께 있을 때만 위자드 트리거
3. **단독 키워드 매칭 폐지**: "일지"나 "보고서"가 단독으로 있으면 트리거하지 않고, 의도를 재확인

**이준호 주임:** 코드 수준에서 보면 이렇게 됩니다:

```javascript
// 개선된 패턴
const reportRequestPattern = /(?:보고서|일지|업무\s*일지).*(?:써줘|쓸래|쓰자|작성|부탁)/;
const negativeContextPattern = /이게\s*.+라고|.+라고\?$|뭐야\s*이게|아닌데|이런\s*게|맞아\?/;

if (negativeContextPattern.test(input)) {
    // 불만 인식 로직으로 분기
} else if (reportRequestPattern.test(input)) {
    // 위자드 시작
}
```

**배성재:** 좋습니다. 여기에 하나 더, **의문형 전체를 한 번 걸러주세요**. "보고서 어디 갔어?", "일지 언제 제출해?" 같은 것도 위자드를 트리거하면 안 되니까요.

**김지은 선임:** 맞습니다. 의문형 필터도 추가하겠습니다.
