# 봉사활동관리 기능 명세서 (이식용 설계도)

> 이 문서는 `xmadx`(PostgreSQL POC)에 구현한 봉사 관련 기능들의 **사양**입니다.
> 실제 운영 EBTS 앱(`poc`, Google Sheets 기반)으로 **이식할 때의 설계도**로 사용하세요.
> 코드는 그대로 복붙 불가(데이터 계층이 PG↔Sheets로 다름) — **규칙·데이터흐름·UI·권한**을 그대로 옮기는 것이 목적.

---

## 0. 공통 데이터 개념
- **지국(주체)** = 사용자가 가입 시 입력한 **소속**(예: 대구지국, 대구남부지국, 대구문화센터). xmadx에선 `users.company_name`(없으면 department/이름)로 그룹핑.
- **가맹점(봉사 대상)** = 봉사 서비스를 받는 대상. xmadx `branches` 테이블에서 `exclude_service=0`. (exclude_service=1 = 물류/본사/공간청춘 등 비대상)
- **참여자** = 봉사활동에 참여한 사람들의 **이름 목록**(완료등록 시 입력). 소속은 등록한 지국으로 귀속.

## 1. 도움말 모드 (hover/탭 안내)
- 헤더에 `?` 토글 버튼. 켜면 `data-help` 속성을 가진 요소에 마우스 오버(또는 모바일 탭) 시 말풍선 안내.
- 구현: `document`에 mouseover/mouseout/focusin + click(capture, 모바일 탭 시 동작 대신 안내) 위임 리스너 1개. 동적/팝업 요소 자동 지원.
- 적용: 로그인/가입, 작성 폼, 하단 네비, 주요 버튼·입력칸에 `data-help="설명"`.

## 2. 지역장 소속 대리수정
- 직책 `지역장`인 사용자가 관리담당자의 **부서·직책·팀** 수정.
- 서버 권한 이중 체크(`position==='지역장'` 또는 admin). 수정은 3개 필드 화이트리스트(비번·회사이동·삭제 불가).
- 제한: **`지역장` 직책 지정·변경은 관리자(감사실/개발자)만** 가능 (지역장 간 권한 부여/강등 차단).

## 3. 워크샵 참석 명단 (엑셀)
- 관리자 전용(개발자 도구). 앱의 관리담당자 목록을 표로 불러오고 추가 인원 더해 엑셀 생성.
- 양식 열: 번호·이름·소속·직함·나이·성별·비고사항 (최소 40행). 나이·성별은 수동 입력.

## 4. 봉사활동 기록 (개인 + 완료등록)
필드: 봉사일자, 활동명, **대상 가맹점**, 장소, 봉사시간, **참여자 명단(이름, 쉼표구분)**, 참여인원(명단 있으면 자동계산), 활동내용, **상태**.
- 상태 4단계: **계획 → 승인 → 완료 → 감사확인**
- 목록/통계: 총 활동·누적시간·올해·이번달. 엑셀 내보내기 지원.

## 5. 상태 워크플로 (버튼 → 상태전환 → 카운터)
| 동작 | 누가 | 전환 |
|------|------|------|
| 계획 작성 / 완료등록 | 지국(작성자 본인) | →계획 / →완료 |
| **승인** | 관리담당자(=지역장/관리자) | 계획→승인 |
| **감사확인** | 감사실/개발자(=관리자) | 완료→감사확인 |
- 검토 화면(지역장/관리자): 전체 지국 항목을 상태별 필터 + 역할에 맞는 버튼 노출.
- 월간 카운터(당월): **계획 / 승인 / 완료 / 감사확인** 4종 집계.

## 6. 가맹점 당월 상태 배지 (목록·검색)
- 봉사 대상 가맹점(exclude_service=0)별로, 당월·전체사용자 활동을 가맹점명 매칭해 상태 산출.
- 우선순위 배지: 🟢승인 > 🟡요청 > 🔵계획, 없으면 ⚪미계획. + **당월 N/2회** 표시.
- 목적: 상세 안 들어가도 "다른 데서 이미 계획/요청/승인했는지" 파악 → 중복 계획 방지.
- 표시: 전국 지국 목록 + 통합검색 지국 결과.

## 7. 봉사 성장 정원 (숨김 점수 → 등급) ★핵심
- **목적**: 참여자/완료 데이터 등록을 유도하는 넛지 장치. 지국이 "어떻게 키우지?" 궁금하게 만든다.
- **점수는 절대 사용자에게 노출 금지.** 백데이터(숨김 테이블)에만 저장하고, 화면엔 **등급(식물)만**.
- **집계 대상**: 상태가 `완료` 또는 `감사확인`인 건만 (완료등록 이상).
- **주체**: 지국(소속) 단위.
- **점수 공식(숨김)** — 지국별 합산:
  - `score = 10·Σ(1/참여인원) + 5·(서로 다른 가맹점 수) + 3·(참여자 이름 중복도)`
    - **1회 참여인원 적을수록 ↑** : 완료건마다 `1/명수` 가산(혼자=1.0, 2명=0.5 …)
    - **서로 다른 가맹점 많을수록 ↑** : 완료한 distinct 가맹점 수
    - **참여자 중복 많을수록 ↑** : 같은 이름이 여러 번 참여한 횟수 합(=Σ max(0, 등장횟수-1))
  - 가중치(10/5/3)·임계값은 한 곳에 상수로 모아 조정 가능하게.
- **등급(임계값, 높은 순)**: 숲(≥100) · 꽃나무(≥60) · 나무(≥30) · 잎(≥12) · 새싹(≥0)
  - 이모지: 새싹🌱 잎🌿 나무🌳 꽃나무🌸 숲🌲
  - **로드맵**: 차후 각 등급 내 세부 종류 추가 → 등급 매핑/임계값만 확장.
- **표시**: 캘린더 상단 패널. 식물 이모지 + 지국명, **무성한 순 정렬**, 상위 3 👑. **수치 일절 없음.**

## 8. 개발자 도구 메뉴 (관리자 전용)
- 워크샵 명단, 봉사 감사확인 진입.

## 이식 시 주의 (PG → Sheets)
- xmadx의 SQL 쿼리/테이블은 Sheets 읽기·쓰기 + 인메모리 집계로 대체.
- 점수 재계산은 완료등록/감사확인 시 또는 정원 조회 시 1회 수행(운영은 단일 인스턴스라 인메모리 캐시 TTL 활용 가능).
- 권한 게이팅(지역장/관리자/감사실)은 운영 앱의 인증 체계에 맞춰 재구현.
- 배포는 운영 규칙 그대로: `gcloud builds submit … :vNNN` → `gcloud run deploy --image …` (2단계).
