# 무지출용팔이 — 포토리얼 3D 사업체 뷰 통합 구조 설계서 v1.0

> 결정(2026-06-12): **포토리얼 목표 · 통합 구조 먼저 · 클라우드판을 GitHub에 올려 직접 통합.**
> 대상 = 라이브 앱(클라우드판: Node/Express + Firestore + Cloud Run, 단일 클라이언트 `public/index.html`).
> 본 문서는 **개별 사업체 씬을 만들기 전, "어디에 어떻게 끼우는가"**를 먼저 확정한다.

---

## 0. 제약 재검토 (중요)
| 항목 | 기존 원칙 | 포토리얼 시 |
|------|-----------|-------------|
| 렌더 | Canvas 2D / 순수 WebGL | **Three.js 필요**(엔진 없이 포토리얼 불가) |
| 외부 라이브러리 | **0** | **3D 뷰에 한해 Three.js 허용**(나머지 게임 로직은 0 유지) |
| 자산 | 코드 내장 도형 | **실제 GLB 모델 + 텍스처**(외부 자산, 라이선스/비용 발생) |
| 로딩 | 단일 HTML 즉시 | **지연 로드**(3D 뷰 열 때만 엔진+자산 fetch) |

→ **핵심 결정**: 게임 본체(루프·경제·인사·결재)는 그대로 두고, **3D는 "사업체 보기" 뷰로 격리·지연로드**한다. 평상시 0 비용, 열 때만 3D.

---

## 1. 전체 아키텍처 (격리·지연로드)
```
[기존 게임 UI/로직]  ──(읽기전용 state)──▶  [Biz3DView 모듈]
   사업체 카드                                  · Three.js 지연 로드(CDN/번들)
   "🏢 3D로 보기" 버튼 ──open(bizId,state)──▶   · 사업체별 GLB 씬 로드
                                                · state→씬 매핑(직원수·레벨·비품)
                                                · 모달/오버레이 <canvas> 렌더
   닫기 ◀──────dispose()─────────────────────  · 닫으면 GPU자원 해제
```
- **단방향 의존**: 3D는 game state를 **읽기만** 한다(게임 로직 수정 0 → 안전).
- **격리**: 3D 코드는 별도 모듈/네임스페이스. 기존 전역과 충돌 없음.

## 2. 폴더 구조 (클라우드판 저장소에 추가)
```
public/
  index.html               ← (수정: 3D 뷰 진입점 + 모듈 로더 1곳만 추가)
  3d/
    biz3d.js               ← Biz3DView 모듈 (열기/닫기/state매핑/카메라/조명)
    loader.js              ← Three.js + GLTFLoader/Draco/KTX2 지연 로드
    scenes/
      office.js            ← IT개발사 개발실 씬 구성(모델 배치·직원 배치 규칙)
      chicken.js           ← 치킨집
      telecom.js           ← 무무통신
      _base.js             ← 공통(바닥/조명/카메라/직원 아바타 배치 유틸)
  assets/3d/
    common/  avatar.glb, props/*.glb   ← 공유 직원 아바타·공통 비품
    office/  room.glb, desk.glb, server_rack.glb, ...
    chicken/ ...
    telecom/ ...
server.js                  ← (확인: express.static('public') 켜져 있어야 자산 서빙)
```

## 3. 모듈 API (확정안)
```js
// public/3d/biz3d.js
window.Biz3D = {
  // 사업체 3D 뷰 열기 (모달/오버레이에 캔버스 마운트)
  open(bizId, snapshot) { /* lazy import three → scene 로드 → 렌더 시작 */ },
  close() { /* 렌더 정지 + scene.dispose + renderer.dispose */ },
  setQuality(level) { /* 'low'|'mid'|'high' — 픽셀비율·그림자·후처리 토글 */ },
};
// snapshot = 게임이 넘기는 읽기전용 사업체 상태(아래 §4)
```
**진입점(기존 index.html에 추가하는 유일한 훅, 추가만):**
```html
<!-- 사업체 카드에 버튼 1개 -->
<button data-biz3d="<bizId>">🏢 3D로 보기</button>
<!-- 하단에 모듈 로더 1줄 -->
<script type="module" src="3d/biz3d.js"></script>
```
버튼 클릭 → `Biz3D.open(bizId, 현재상태)` 호출. 기존 로직은 손대지 않음.

## 4. 데이터 → 씬 매핑 (state-driven)
게임 상태가 3D 장면을 결정한다(같은 사업체라도 성장하면 달라짐).
| 게임 state | 3D 반영 |
|------------|---------|
| `staffCount` (직원 수) | 책상에 앉은 **아바타 수** |
| 직원 `level`/직급 | 아바타 복장·자리(팀장석 등) |
| 사업체 `level`/업그레이드 | 비품 등급(모니터 수·서버랙·인테리어 티어) |
| `reputation`/매출 | 조명·청결·디테일(화분·트로피·간판) |
| 업종(`category`) | 로드할 씬(office/chicken/telecom…) |
> snapshot 스키마는 클라우드판 실제 state 확인 후 확정(§8 오픈이슈).

## 5. 자산 파이프라인 & 소싱 (포토리얼의 핵심·정직한 비용)
- **포맷**: glTF **GLB** + **Draco**(지오메트리 압축) + **KTX2/Basis**(GPU 텍스처 압축) → 모바일 용량·메모리 최적화.
- **소싱(라이선스 주의)**:
  - 무료: Poly Haven(CC0), Quaternius, Kenney(스타일라이즈드), Sketchfab CC.
  - 유료/의뢰: 포토리얼 사무실·매장 키트(상업 라이선스 확인 필수).
  - ⚠️ **포토리얼 = 외부 자산 의존**(시간·비용·라이선스). 이 부분은 코드만으로 안 되고 자산 확보가 병행돼야 함.
- **용량 예산**: 사업체당 GLB+텍스처 **≤ 3~5MB**(Draco/KTX2 후) 목표. 지연로드 + 캐시.

## 6. 성능 / 모바일 / 폴백 (QA 원칙 유지)
- **지연 로드**: 3D 뷰 열 때만 Three.js(+자산) fetch. 초기 앱 무게 0.
- **품질 자동조절**: `devicePixelRatio` 캡, 그림자/후처리 on-off, 저사양 자동 'low'.
- **폴백 사다리**: 포토리얼(고) → 스타일라이즈드 WebGL(중, 이미 보유: `poc-3d-office.html`) → 2D(저).
- **정리**: 닫을 때 `dispose()`로 GPU 메모리 해제(누수 방지).

## 7. 단계 로드맵 (통합 구조 먼저)
| 단계 | 산출 | 비고 |
|------|------|------|
| **P0 통합 골격** | `biz3d.js` 로더 + 모달 + Three.js 지연로드 + **플레이스홀더 박스 씬**이 사업체 버튼으로 열림 | **자산 0으로 구조부터 검증** ← 지금 먼저 |
| P1 첫 포토리얼 | IT개발사 office GLB 1세트 + state매핑(직원수) | 자산 확보 후 |
| P2 업종 확장 | 치킨집·무무통신… 씬 추가 | 업종 1개씩 |
| P3 상태연동 심화 | 레벨·평판·업그레이드 → 비품/인테리어 티어 | 폴리시 |

> P0는 **자산 없이도** 가능(플레이스홀더). 구조가 맞는지부터 확정 → 그 위에 자산만 갈아끼움.

## 8. 배포 고려 (Cloud Run)
- 자산은 `public/assets/3d/`에 두고 `express.static` 서빙(또는 차후 CDN/GCS 분리).
- GLB/KTX2에 **장기 캐시 헤더** 권장(`Cache-Control: immutable`).
- 컨테이너 이미지 비대화 주의 → 자산이 커지면 **GCS 버킷 + CDN**으로 분리 검토.
